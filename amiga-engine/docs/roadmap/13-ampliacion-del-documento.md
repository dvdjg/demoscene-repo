# Cómo ampliar esta referencia

## Convenciones de nombres

- Páginas numeradas `NN-tema.md` para mantener un orden de lectura sugerido.
- Páginas no numeradas para anexos puntuales: `anexo-nombre.md` o subdirectorios por tema (`roadmap/agl/`, `roadmap/ocs/`).

## Qué añadir en cada nueva página

1. **Contexto** — a qué capa pertenece (alto/bajo).
2. **Estado actual** — implementado / parcial / solo diseño (actualizar cuando cambie el código).
3. **Referencias en código** — rutas a headers y módulos.
4. **Pruebas** — cómo verificar cambios.
5. **Enlaces** — anterior/siguiente o vuelta al [índice](README.md).

## Diagramas

Mermaid u otros diagramas son bienvenidos en páginas dedicadas para no recargar el índice.

## Sincronización con el roadmap ejecutable

Si el equipo usa issues o milestones externos, enlázalos desde `03-hitos-fases.md` o desde esta página en una tabla “Milestone → documentos”.
